# 本科毕业论文写作要素总结

> 论文规模：正文 35 页 / 全文 45 页；参考文献 30 篇；图 16 张；表 2 张；公式 34 个

---

## 一、整体结构（深北莫工科本科论文标准模板）

固定的前置/后置部分：

1. **封面**：中英文标题 + 姓名 / 院系 / 专业 / 学号 / 指导教师 / 职称 / 提交日期
2. **诚信声明 + 学位论文使用授权书**（学校统一模板，签名）
3. **中文目录** + **English Contents**（双语目录）
4. **中文摘要 + 关键词**（5 个关键词，分号分隔）
5. **英文 Abstract + Keywords**（与中文摘要逐句对应）
6. **正文章节**（5 章为主流结构）
7. **参考文献**
8. **致谢**

页眉：学校 logo；页脚："第 X 页 共 45 页"。

---

## 二、正文五章式骨架（强烈推荐套用）

| 章 | 标题 | 篇幅占比 | 核心任务 |
|---|---|---|---|
| 1 | **引言** | ~5% | 1.1 研究背景和意义；1.2 研究目标 + 论文结构安排 |
| 2 | **相关研究** | ~15% | 综述本研究用到的每一项关键理论/技术，按子方向分小节（2.1~2.5），每节 1~2 页 |
| 3 | **系统设计与框架** | ~15% | 3.1 总体架构概述 + 总体框架流程图；3.2~3.N 各核心模块的设计思路（不写实现细节） |
| 4 | **系统实现与实验结果** | ~50% | 4.1~4.3 各模块的具体实现（数据集、网络结构、公式、参数）；4.4 实验与结果分析 |
| 5 | **总结与展望** | ~5% | 5.1 总结（重述贡献）；5.2 未来展望（局限 + 后续方向） |

**第三章 vs 第四章的分工**——这是新手最容易混的点：
- 第三章只画**框图、讲思路、说"为什么这样设计"**；
- 第四章才上**数据集名称、超参数、网络层细节、公式编号、训练曲线、表格**。

---

## 三、摘要写作要素（中英对应）

按"问题 → 方法 → 创新点 → 实验结论 → 应用价值"五段递进：

1. **问题陈述**："针对当前 X 中存在的 …… 问题"
2. **总方法**："提出了一种基于 …… 的 X 框架，旨在 ……"
3. **方法分解**：分别交代 2~3 个核心模块各自做了什么（融合机制、双层调节、记忆机制 …）
4. **实验结论**："实验表明 …… 显著降低了 …… 方差 p<0.05 …… 避免了 …… 问题"
5. **价值落地**："为 X、Y 等场景提供了兼具鲁棒性与人性化的解决方案"

关键词：3~5 个，覆盖"领域 + 核心方法 + 模型 + 应用"。

---

## 四、引言段落组织模式

**1.1 研究背景和意义** 推荐 3~4 段递进：
- 第 1 段：领域整体重要性（情感是人机交互不可或缺的部分 …）
- 第 2 段：领域近年发展趋势 + 现存局限（情感计算从静态分类走向动态多模态 …）
- 第 3 段：本课题选择该切入点的现实意义（桌面级机器人 + 应用前景）

**1.2 研究目标** 必须包含：
- 研究总目标一句话；
- 按"模块 1 / 模块 2 / 模块 3"分别说明各自要解决什么；
- **末段固定模板**："本论文的主要结构安排如下：第一章为 ……；第二章 ……；第三章 ……；第四章 ……；第五章 ……。"

---

## 五、相关研究（文献综述）写作模式

**每个 2.x 子节的统一段落结构**：

1. 引出问题/背景一句话（"与视觉模态不同，语音识别需要 ……"）
2. 介绍该理论/模型的核心机制 + 提出者（Mehrabian / Plutchik / LeCun … + [n] 引用）
3. 罗列其代表性变体或扩展（Neural Turing Machines [23]、Memory Network [24] …）
4. **回到本研究**："因此，本研究在 …… 模块中采用了 …… 架构"——把综述和自己用的方法挂钩

每个引用必须方括号编号 [n]，编号顺序就是文中首次出现的顺序（GB/T 7714 / IEEE 风格混合）。

---

## 六、系统设计章节（第 3 章）写法

- **3.1 总体架构概述**：一段话讲三模块协同 → 给一张总框架流程图（图 4） → 一段话再描述图中每个模块的输入/输出/串接关系
- **3.2~3.N 模块设计**：
  - 一句"核心功能定位"
  - 列出该模块包含的子组件
  - 给该模块的流程图（每个模块至少一张图）
  - 文字描述各子组件作用 + 选择该方案的理由

特点：**只讲 what + why，不讲 how**。例如说"采用基于置信度的融合策略"，但不写置信度公式——公式留给第 4 章。

---

## 七、系统实现章节（第 4 章）的写作要素

每一个 4.x.y 实现小节，标准结构是：

1. **数据集介绍**：名称、规模、类别数、采样率/分辨率、训练-测试划分
   > 例："本研究采用 FER2013 数据集 …… 包含 35,887 张图像 …… 七种基本情绪 …… 48×48 像素"
2. **特征提取/预处理**：列点说明（MFCC / 色度特征 / 梅尔频谱 / 频谱对比度）
3. **模型架构**：
   - 文字描述层数、滤波器数、核大小、激活函数
   - 给出网络结构图（图 5、图 6）
   - 关键代码风格的层级展示（`nn.Linear(512, 256)`、`nn.Dropout(0.5)` …）
4. **训练策略**：优化器、数据增强、学习率
5. **结果可视化**：训练 loss/accuracy 曲线图（图 12、图 13）+ 指标解读

**4.4 实验与结果分析** 的写法：
- 设计一个完整模拟场景（70 秒 / 每 5 秒采样一次）
- 给原始数据表（表 1 人脸情绪、表 2 语音情绪）
- 给融合结果对照表（图 15）
- 给最终趋势曲线图（图 16），区分"应激情绪 / 调整后情绪 / 用户情绪"多条曲线
- 文字按"step-by-step 描述结果 + 揭示规律 + 验证假设"展开

## 八、公式与图表写作规范

### 公式
- 编号格式 `(章号.序号)`，右对齐，如 `(4.13)`
- 多步推导分行编号（4.9 → 4.10 → 4.11 …）
- 公式后立即解释每个符号：
  > "其中 $P_{base}$、$A_{base}$、$D_{base}$ 表示第一层的应激情绪 PAD 值"
- 范围限制、归一化、加权融合是工科论文常用公式形态

### 图
- 编号 `图 N` + 居中标题
- 必含图：总框架图、各模块流程图、网络结构图、训练曲线图、最终结果对比图
- 颜色策略：模块用不同色块 + 箭头区分数据流 / 控制流

### 表
- 编号 `表 N` + 居中标题
- 三线表风格（顶线、表头线、底线）
- 实验结果表配套一段"根据表 N 数据分析可知 ……"的解读

## 九、语言风格特征

1. **学术化但不晦涩**：长句多用"通过…实现…，从而…"的递进结构
2. **第一人称使用"本研究"/"本框架"**，少用"我"，偶尔用"我们"
3. **每段第一句往往是主题句**（topic sentence），先抛结论再展开
4. **承上启下短句**：
   - "在此背景下 ……"
   - "基于此理论框架 ……"
   - "综上 ……"
   - "如图 N 所示 ……"
5. **避免空泛形容词**，用量化语言："准确率达到 60%"、"方差降低 p<0.05"、"损失收敛至 0.6 以下"
6. 中英文混排时，专业术语首次出现给出全称 + 缩写：
   > "情感计算（Affective Computing）[3][4]"、"Pleasure-Arousal-Dominance, PAD"

## 十、参考文献规范

- 数量：本科约 **20~30 篇**（本文 30 篇）
- 类型分布：**期刊论文 + 顶会论文 + arXiv 预印本 + 经典专著**混合
- 时间分布：经典文献（1980s-2000s）+ 近 5 年文献（保证前沿性）
- 格式：作者、标题、期刊/会议、卷期号、页码、年份、DOI/链接
- 在正文中按出现顺序编号 [n]，同一处可串引 `[15][16][17]`

---

## 十一、致谢写作公式

四段法：

1. **导师致谢**（最详细，约半页）：学术指导 + 资源支持 + 治学态度 + 对自己的影响
2. **协作者致谢**（同学/同事的具体帮助：数据分析、答辩准备）
3. **亲友致谢**（陪伴与精神支持）
4. **评审专家 + 兜底感谢**（"未能一一提及的所有人士"）

---

## 十二、本范文可借鉴的"加分技巧"

1. **双语目录**——满足学校国际化要求，提升正式感
2. **总框架图放在第 3 章开头**，让读者快速建立全局视图
3. **每个模块都有独立流程图**——视觉化降低阅读成本
4. **方差减少百分比公式注释**单独标注在图下，增强实验严谨性
5. **未来展望具体到下一步可执行的技术路线**（OpenVLA 集成），不是空泛口号
6. **5.1 总结部分**重新组织语言重述贡献，**不是摘要的复制粘贴**
7. **数据集 → 模型 → 训练 → 结果 → 分析**的工科叙事链条完整闭环

## 十三、写作前必备清单（自查 checklist）

- [ ] 中英文标题一致、关键词覆盖核心方法
- [ ] 摘要符合"问题-方法-创新-结果-价值"五段
- [ ] 引言末段交代论文章节安排
- [ ] 第 2 章每个理论小节都"挂钩"到自己的方案
- [ ] 第 3 章只讲设计、第 4 章只讲实现，不重复
- [ ] 每个模块至少一张流程图 / 网络图
- [ ] 公式编号、符号说明齐全
- [ ] 实验有数据集来源、超参数、训练曲线、对比表、结果解读
- [ ] 总结部分点出方差量化提升
- [ ] 展望部分有可执行的下一步路线
- [ ] 参考文献 ≥20 篇，含近 3 年文献
- [ ] 致谢覆盖"导师 / 协作者 / 亲友 / 评审"四类对象
