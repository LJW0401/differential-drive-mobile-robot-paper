## 项目概述

### 研究目标

构建一套仿真与现实融合的轮腿机器人控制系统：用户在 MuJoCo 仿真环境中操控机器人，实物机器人同步执行对应动作。在此基础上，对仿真与实物的运动响应进行定量对比分析，验证 Sim-to-Real 迁移的可行性。

### 技术路线

1. 在 MuJoCo 中搭建五连杆并联腿轮腿机器人仿真模型
2. 设计并实现底层运动控制算法（关节 PID / 逆运动学）
3. 基于 Jetson Orin Nano 搭建硬件通信链路（CAN 总线）
4. 实现仿真到实物的指令传递与协同控制（方案 A）
5. 采集仿真与实物对比数据，进行 Sim-to-Real 一致性评估
6. （如有余力）升级为双向数字孪生：实物状态实时反馈至仿真（方案 B）

### 先导研究基础

创新实践课已完成的工作（2025 年 12 月）：

- 基于 MuJoCo 的四轮差速机器人仿真模型搭建
- PID 轮速控制算法实现与仿真验证
- 仿真与实物系统的动态响应、稳态误差、控制力矩对比分析
- 验证了仿真中整定的 PID 参数可直接迁移至实物

本次毕设在此基础上，将机器人本体从简易差速底盘升级为五连杆轮腿结构，并实现仿真与实物的实时协同控制。

## 系统架构

### 方案 A：仿真作为遥控前端（核心目标）

```
┌─────────────────── Jetson Orin Nano ───────────────────┐
│                                                         │
│  ┌──────────┐   ZMQ/共享内存   ┌──────────────────┐    │
│  │ MuJoCo   │ ──────────────→ │ 控制进程          │    │
│  │ 仿真界面  │  目标关节角度/    │ (关节PID/逆运动学) │    │
│  │ (用户操作) │  目标速度        │                  │    │
│  └──────────┘                  └────────┬─────────┘    │
│                                         │ SocketCAN    │
│                                         ▼              │
│                                  ┌──────────────┐      │
│                                  │ CAN 收发器    │      │
│                                  └──────┬───────┘      │
└─────────────────────────────────────────┼──────────────┘
                                          │ CAN Bus
                                          ▼
                                   ┌──────────────┐
                                   │ 电机驱动器     │
                                   │ (各关节电机)   │
                                   └──────────────┘
```

### 方案 B：双向数字孪生（进阶目标）

在方案 A 基础上增加反向链路：

- 实物机器人的编码器、IMU 等传感器数据通过 CAN 总线回传
- 控制进程将实物状态同步更新到 MuJoCo 仿真中
- 仿真界面实时显示实物的真实状态，可用于状态预测与异常检测

## 关键技术

| 模块 | 技术方案 | 说明 |
|------|---------|------|
| 仿真引擎 | MuJoCo | 高精度物理仿真，Python 接口友好，适合 ARM64 |
| 机器人建模 | MJCF (XML) | 五连杆并联闭链结构，需处理运动学约束 |
| 底层控制 | PID + 逆运动学 | 各关节独立 PID，通过虚拟腿模型映射到关节空间 |
| 进程间通信 | ZMQ (pyzmq) | 仿真进程与控制进程之间的低延迟消息传递 |
| 硬件通信 | SocketCAN + python-can | Linux 原生 CAN 总线框架，Jetson Orin Nano 硬件支持 |
| 计算平台 | Jetson Orin Nano | 仿真与控制算法均在此运行 |
| 评估方法 | 阶跃响应 / 稳态误差 / 轨迹对比 | 复用先导研究的评估指标体系 |

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 五连杆闭链建模困难 | 仿真模型不准确 | 先用简化模型验证，逐步增加复杂度；参考 MuJoCo 官方闭链约束文档 |
| CAN 通信调试耗时 | 硬件链路延迟 | 提前获取电机驱动协议文档；先单电机调通再扩展到全部关节 |
| Jetson 算力不足 | 仿真帧率低 | 降低仿真渲染精度；仿真与渲染分离，控制回路优先保证实时性 |
| 仿真与实物差距大 | Sim-to-Real 效果差 | 系统辨识调整仿真参数；在论文中如实分析差异原因 |
| 时间不足以完成方案B | 功能不完整 | 方案B定位为加分项，方案A足以支撑毕设 |
