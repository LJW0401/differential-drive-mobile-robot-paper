# 第一章 引言

## 1.1 研究背景和意义

近年来，随着自动化与人工智能技术的不断进步，移动机器人在工业生产、仓储物流、室内巡检、商业服务以及科研教学等场景中得到日益广泛的应用。在众多移动机器人构型中，差速驱动（Differential-Drive）结构以其机械结构紧凑、传动链短、控制变量少、零半径转向能力强、制造与维护成本相对较低等突出优点，长期以来被作为轮式移动机器人研究的代表性对象 [1][6]。从早期的实验室原型到今天的商用扫地机器人、AGV 与服务机器人底盘，差速驱动机器人始终是连接控制理论研究与工程实现之间的重要载体。对于此类机器人而言，运动控制性能直接决定了路径跟踪精度、姿态稳定性以及任务执行效率，因此围绕其运动建模、仿真验证与控制算法实现开展系统性研究，具有较强的工程价值和学术意义。

差速驱动移动机器人的控制问题涉及非线性运动学、非完整约束以及左右轮耦合等多重特性，单纯依赖实物平台进行算法迭代往往面临开发周期长、调试风险高、可重复性差等问题。为此，业界普遍采用"先仿真、后实物"的研发范式，通过物理仿真平台在不损伤硬件的前提下完成模型构建、参数整定和控制策略验证。MuJoCo、Gazebo、Webots 等仿真工具是当前主流选择，其中 MuJoCo 凭借其高效的物理求解器、灵活的建模描述以及对控制接口的良好支持，越来越多地被用于机器人控制算法的快速迭代研究 [5]。在控制算法方面，PID 控制因结构简单、易于工程实现，仍是移动机器人底层控制中最常用的方法；LQR 控制基于状态空间建模与二次型代价函数优化，能够在系统稳定性与控制代价之间取得较优折中；MPC（Model Predictive Control，模型预测控制）则可以显式处理系统约束，在轨迹跟踪与多变量控制场景中表现出明显优势 [3][4]。然而，已有研究多聚焦于单一控制方法的改进，较少在统一对象、统一仿真平台和统一评价指标下对 PID、LQR、MPC 进行系统化对比，且仿真验证与实物部署之间的衔接环节仍缺乏完整的工程示例，给本科教学与工程入门带来一定门槛。

针对上述问题，本课题以差速驱动移动机器人为研究对象，构建了一条"运动建模 → MuJoCo 仿真 → 多算法控制 → 实物部署"的完整技术链路，重点关注三类典型控制方法在同一对象上的对比与迁移。通过建立可复用的仿真平台和统一的实验工况，本研究一方面可以为差速驱动机器人控制系统的设计与调参提供更系统的实验依据，另一方面也可作为后续移动机器人控制研究的基础平台，便于在其上进一步开展感知融合、轨迹规划以及基于学习的控制等扩展研究。课题所形成的方案对室内物流配送、自主巡检、教学实验平台搭建等工程场景具有一定的参考价值，亦能够为本科阶段开展机器人控制研究提供具有工程闭环的范式示例。

## 1.2 研究目标与论文结构

围绕差速驱动移动机器人的运动仿真与控制研究，本课题的总体研究目标是：建立一套从理论建模到实物验证完整闭环的研究方案，并在统一的仿真与实验平台上完成 PID、LQR 与 MPC 三类典型控制方法的实现与对比分析，为差速驱动移动机器人控制系统的设计与优化提供实验依据。

为实现上述总目标，本研究将其拆分为以下四个相互衔接的子目标：

1. **建模目标**：基于差速驱动结构的几何关系与受力分析，建立机器人的运动学与动力学模型，明确状态变量、控制输入及非完整约束的表达形式，为仿真模型搭建与控制器设计提供统一的数学基础；
2. **仿真平台目标**：基于 MuJoCo 物理仿真引擎，构建包含机器人模型、运动场景与统一控制接口的仿真平台，确保三类控制器能够在同一环境、同一参数和同一工况下进行测试与对比，提升实验结果的可比性和可复现性；
3. **控制算法目标**：分别实现 PID、LQR 与 MPC 三类控制器，覆盖经典反馈控制、最优控制和预测控制三种典型范式，并在直线运动、定点转向和轨迹跟踪等典型任务下完成参数整定与性能评估；
4. **实物迁移目标**：将仿真平台中验证通过的控制逻辑迁移到差速驱动机器人实物平台上，完成基本的运动控制与轨迹跟踪实验，分析仿真与实物之间的差异及其产生原因，验证整套技术链路的可行性。

**本论文的主要结构安排如下**：第一章为引言，介绍课题的研究背景、意义、研究目标及论文结构；第二章为相关研究，系统综述差速驱动机器人建模方法、主流仿真平台以及 PID、LQR、MPC 三类控制算法的国内外发展现状，并将其与本研究方案相挂钩；第三章为系统设计与框架，给出本系统的总体架构以及建模、仿真平台、控制算法和实物控制系统四个模块的设计思路；第四章为系统实现与实验结果，详细描述各模块的具体实现方式、关键公式与参数选取，并通过典型工况下的仿真和实物实验对三类控制算法进行量化对比分析；第五章为总结与展望，回顾本文主要工作与创新点，分析当前方案的不足，并对未来可拓展的研究方向进行展望。

# 第二章 相关研究

差速驱动移动机器人的运动仿真与控制研究涉及运动学/动力学建模、物理仿真平台以及多种典型控制算法等多个方面。本章按照"建模方法 → 仿真平台 → 控制算法"的脉络，对国内外相关研究进行综述，并在每一小节末尾说明本研究在该方向上的方法选择，以便后续章节中的具体设计与实现有据可循。

## 2.1 差速驱动移动机器人运动学与动力学建模

差速驱动机器人通过左右两个独立驱动轮的转速差实现前进与转向，其运动具有典型的非完整约束（nonholonomic constraint）特性，即车体在垂直于轮轴方向的速度恒为零，这一约束无法通过状态变量本身的代数关系消去，给运动建模与控制器设计带来了一定难度 [1]。运动学建模通常以车体位姿 $(x, y, \theta)$ 为状态量，以左右轮线速度 $v_l, v_r$ 或等效的车体线速度 $v$ 与角速度 $\omega$ 为输入，建立刚体平面运动方程，从而刻画机器人在世界坐标系下的位姿演化规律。DeSantis 较早地从路径跟踪角度系统地研究了差速驱动结构的运动学建模与控制问题，给出了基于位姿误差的非线性反馈控制框架，奠定了后续大量轮式机器人路径跟踪研究的基础 [1]。

在运动学模型之上，进一步考虑机器人质量、转动惯量、轮端摩擦与电机驱动特性，可得到差速驱动机器人的动力学模型。Kumar 等人在轮式移动机器人的建模与轨迹跟踪研究中，综合考虑了车体平动、转动以及轮地接触力的影响，建立了二阶动力学方程，并指出动力学模型对于实现高动态、高精度控制具有重要意义 [6]。Alves 等人则进一步从工程实现角度出发，针对差速驱动机器人开展了参数辨识与控制器设计的联合研究，通过实验数据辨识机器人质量、阻尼与电机时间常数等关键参数，使所建立的动力学模型更加贴近实物平台 [2]。这些研究表明，运动学模型适合用于轨迹规划和高层路径跟踪控制，而动力学模型则在涉及加速度约束、力矩规划及高速运动的场景中具有不可替代的作用。

针对建模方法的选择，已有研究中也出现了若干代表性变体：一类是以极坐标形式表示位姿误差，将路径跟踪问题转化为极径与方位角的耦合控制 [3]；另一类是基于扰动观测器或模糊补偿的鲁棒建模思路，用以处理打滑、负载变化等不确定性。考虑到本研究的实验对象为典型的轮式差速驱动平台，且控制目标集中在直线、转向与轨迹跟踪等基础任务，本研究在第 4 章中将以经典笛卡尔坐标下的运动学方程作为统一建模基础，并在动力学层面参考 Alves 等人的简化模型 [2]，重点保留质量、转动惯量与轮端力矩三类核心要素，以便在保证物理真实性的同时，使后续 LQR 与 MPC 控制器易于线性化与离散化处理。

## 2.2 移动机器人仿真平台

仿真平台是移动机器人控制算法研究的重要支撑工具，其作用主要体现在三个方面：一是提供可重复的实验环境以隔离硬件不确定性，二是缩短控制算法迭代周期并降低实物调试风险，三是为不同控制方法提供统一的对比测试基础。当前主流的机器人仿真平台主要包括 Gazebo、Webots 与 MuJoCo 等。Gazebo 与 ROS 生态深度集成，适合开展整机系统级仿真，但其物理求解步长较大、控制接口相对厚重，在高频闭环控制研究中存在一定限制；Webots 则在教育与轻量级机器人仿真中应用较多，但场景复杂度提升时其求解效率会有所下降。

MuJoCo（Multi-Joint dynamics with Contact）由 Todorov 等人提出，是一款专为基于模型的控制研究设计的物理仿真引擎 [5]。其在接触建模、关节求解和数值稳定性方面进行了专门优化，支持小步长高频仿真，并提供了简洁的 MJCF（MuJoCo XML）模型描述格式与多语言绑定接口（C/C++、Python 等），使研究者可以便捷地将仿真器作为控制回路的一部分。在差速驱动等轮式机器人控制算法研究场景下，MuJoCo 既能够稳定地仿真轮地接触与摩擦特性，又能够以较高的实时倍率运行，从而支持 PID、LQR、MPC 等多种控制方法在同一平台下的快速迭代与对比验证 [5]。

近年来，将 MuJoCo 用于轮式机器人控制研究的工作不断增多，研究者通常以 MJCF 描述车体几何与质量参数，借助传感器与执行器接口暴露状态量与控制输入，并结合 Python 控制脚本完成闭环仿真。这一组合具有"建模轻、迭代快、接口清晰"的特点，与本研究"统一仿真、多算法对比"的目标契合度较高。综合考虑求解效率、控制接口灵活性以及与 PID/LQR/MPC 算法集成的便捷性，本研究选择 MuJoCo 作为差速驱动移动机器人仿真平台的核心引擎，并在第 3 章中给出基于 MJCF 的机器人模型与统一控制接口设计。

## 2.3 PID 控制方法及其在移动机器人中的应用

比例-积分-微分（Proportional-Integral-Derivative, PID）控制由经典控制理论发展而来，凭借结构简单、参数物理意义明确、调参经验成熟等优势，长期以来是工业控制和移动机器人底层控制中应用最广泛的方法之一。在差速驱动机器人中，PID 控制通常被部署在两类典型回路：一是轮速或电机转速闭环，用于跟踪上层期望的左右轮速度；二是车体位姿与速度闭环，用于在更高层次实现直线行驶、定点转向以及简单的路径跟踪。

针对差速驱动机器人的 PID 应用研究中，Alves 等人在参数辨识所得动力学模型基础上设计了串级 PID 控制器，分别完成速度环与位置环的闭环调节，并通过实验验证了其在跟踪精度与抗扰动能力方面的有效性 [2]。Kumar 等人在轮式机器人轨迹跟踪研究中，将 PID 与运动学模型结合，提出了基于位姿误差反馈的双环 PID 控制结构，证明了在小偏差线性化条件下其良好的工程适用性 [6]。已有研究还对 PID 进行了多种扩展，如引入前馈补偿、抗积分饱和、自适应或模糊整定等，以增强其在非线性与变工况场景下的鲁棒性。

PID 控制的局限主要表现在两个方面：其一，PID 是一种典型的单输入单输出（SISO）控制范式，对于差速驱动机器人这种存在左右轮耦合的多输入系统，往往需要分通道独立调节，难以充分利用系统模型信息；其二，对于带显式约束（如轮速饱和、加速度限制）的轨迹跟踪任务，PID 缺乏显式的约束处理机制，必须依赖外部限幅或抗饱和补偿。综合考虑工程实现难度与作为对照基线的代表性，本研究在第 3 章中将 PID 设计为速度环+姿态环的双环结构，并以其作为后续 LQR 与 MPC 算法对比分析的参考基线。

## 2.4 LQR 控制方法

线性二次型调节器（Linear Quadratic Regulator, LQR）是现代控制理论中的经典方法，其核心思想是在线性时不变系统的状态空间模型基础上，通过最小化二次型代价函数 $J = \int (x^T Q x + u^T R u)\,dt$ 求解最优状态反馈律，从而在系统稳定性与控制代价之间取得最优折中。LQR 求解过程归结为代数 Riccati 方程的求解，所得反馈增益矩阵 $K$ 具有解析的最优性，并可通过权重矩阵 $Q, R$ 的调节体现设计者对状态收敛速度与控制能耗的不同偏好。

将 LQR 应用于差速驱动移动机器人轨迹跟踪是近年来的常见思路。由于差速驱动模型本身是非线性的，研究者通常先以期望轨迹为参考状态，对运动学/动力学模型在工作点附近进行线性化，从而得到适合 LQR 设计的近似线性时变（LTV）系统。Cornejo 等人针对差速轮式移动机器人的轨迹跟踪问题，对极坐标控制和 LQR 两种方法进行了系统对比，结果表明 LQR 在跟踪精度和控制平滑性方面均优于经典极坐标控制，且更便于通过调节 $Q, R$ 体现设计目标 [3]。Alves 等人在其参数辨识基础上也讨论了线性最优控制方法在差速驱动机器人上的可行性，进一步支持了 LQR 作为对照算法的合理性 [2]。

围绕 LQR 在轮式机器人上的应用，已有多种代表性扩展，例如：基于轨迹线性化的迭代 LQR（iLQR）以处理非线性效应、引入状态估计的 LQG 框架以应对量测噪声、以及结合积分作用的 LQI 控制以消除稳态误差等。考虑到本研究的目标是在统一对象上完成 PID、LQR、MPC 三类典型控制范式的对比分析，过于复杂的 LQR 变体并非重点。本研究在第 4 章中将围绕差速驱动机器人位姿误差的线性化模型，构建标准状态空间形式的 LQR 控制器，重点关注 $Q, R$ 权重的工程整定方法及其对响应特性的影响。

## 2.5 模型预测控制（MPC）方法

模型预测控制（Model Predictive Control, MPC）是一类基于显式系统模型、滚动优化和反馈校正的控制方法。其基本思想是在每个控制周期内，依据当前状态和预测模型，在有限预测时域内求解一个带约束的优化问题，得到最优控制序列后仅执行第一项控制输入，并在下一个周期重复上述过程。相比于 PID 与 LQR，MPC 的最大特点在于能够显式考虑状态、输入及其变化率的约束条件，因此在涉及饱和、避障、动力学受限的轨迹跟踪问题中具有天然优势 [4]。

针对差速驱动移动机器人的 MPC 研究近年来发展迅速。Bouzoualegh 等人针对差速驱动机器人轨迹跟踪问题，建立了离散化预测模型，并基于二次规划（QP）求解器实现了线性 MPC 控制，仿真结果表明 MPC 在跟踪精度、抗扰动性以及对轮速约束的处理上均优于经典反馈控制 [4]。Kumar 等人也在轮式机器人轨迹跟踪综述与建模工作中将预测控制视为差速驱动机器人未来的重要方向之一 [6]。在算法形式上，针对差速驱动机器人的 MPC 主要包括两类：一类是以线性化模型为基础的线性 MPC（LMPC），通过 QP 实时求解，工程实现成熟；另一类是基于非线性模型的非线性 MPC（NMPC），具有更高的精度但实时性挑战较大，常借助 CasADi、acados 等工具进行求解。

在求解器层面，OSQP 等开源 QP 求解器因其求解速度快、对热启动支持友好，已成为线性 MPC 工程实现中的常见选择，使 MPC 控制器可以在嵌入式平台与机器人控制频率（如 50–100 Hz）下稳定运行。考虑到本研究关注控制效果的对比分析而非求解器的极致优化，本课题在第 4 章中将围绕差速驱动机器人的运动学线性化模型构建线性 MPC 控制器，预测时域和控制时域取适中规模，约束包括轮速上限与加速度上限，求解器采用工程上较为成熟的 QP 方案，以便与 PID、LQR 控制器在统一仿真平台下完成横向对比。

## 2.6 本章小结

本章围绕差速驱动移动机器人的运动仿真与控制研究，从建模方法、仿真平台到控制算法三个层面对国内外相关工作进行了梳理。在建模方面，已有研究在运动学与动力学模型方面均较为成熟，本研究将采用经典的笛卡尔坐标运动学模型并结合简化动力学模型作为统一建模基础；在仿真平台方面，MuJoCo 凭借其求解效率和接口灵活性，成为多算法对比研究的合适选择；在控制算法方面，PID、LQR 与 MPC 分别代表了经典反馈控制、最优控制与预测控制三种典型范式，三者共同覆盖了从工程基线到先进控制的主要设计思路。基于以上综述，本研究将在第 3 章中给出涵盖建模、仿真、多算法控制以及实物迁移的系统总体设计。

# 第三章 系统设计与框架

本章在第 2 章相关研究的基础上，围绕"运动建模 → 仿真验证 → 多算法控制 → 实物迁移"的总体研究目标，给出本课题的系统总体架构和各模块的设计思路。本章只关注模块的功能定位、组成结构以及设计理由（即"做什么"与"为什么"），具体的数学推导、参数设定与代码实现细节在第 4 章中给出。

## 3.1 总体架构概述

为了在统一的实验环境下完成 PID、LQR 与 MPC 三类典型控制方法的对比，并确保仿真验证后的控制逻辑能够顺利迁移到实物平台，本研究将整体系统划分为四个相互衔接的模块：**机器人建模模块**、**MuJoCo 仿真平台模块**、**控制算法模块**和**实物控制系统模块**。其中，机器人建模模块负责给出统一的数学描述与状态-输入定义；仿真平台模块基于该数学模型构建 MuJoCo 中的物理实体与运行环境；控制算法模块在仿真环境提供的状态接口之上，分别实现三类控制器；实物控制系统模块则将仿真验证通过的控制逻辑迁移到差速驱动机器人实物平台，完成最终的工程实现与验证。

**图 1** 系统总体框架图

如图 1 所示，整套系统遵循"模型驱动 + 仿真验证 + 实物部署"的纵向流程：建模模块的输出（运动学/动力学方程、状态变量定义）同时作为仿真平台与控制算法模块的输入；仿真平台向控制算法模块提供机器人当前状态并接收控制指令，构成闭环；控制算法模块对外暴露统一的控制接口，使 PID、LQR、MPC 三种控制器可以在不修改仿真环境的前提下相互替换；最终，仿真中验证后的控制策略以参数文件与控制脚本的形式迁移到实物控制系统，由其完成上层期望速度到底层电机驱动的转换。这一架构的核心特点是**接口统一**与**逻辑可迁移**：所有控制器面对相同的状态/输入定义，避免因接口差异引入额外的对比偏差；同时仿真与实物共享一致的控制律实现，便于分析两者之间的差异来源。

## 3.2 差速驱动机器人建模模块设计

差速驱动机器人建模模块是整个系统的数学基础，其核心功能定位是：以统一的状态变量、控制输入和动力学描述，为仿真平台搭建与控制器设计提供共同的数学模型。该模块主要包含三个子组件：**运动学模型**、**动力学模型**和**统一状态-输入定义**。运动学模型描述位姿 $(x, y, \theta)$ 与车体线/角速度之间的微分关系，用于轨迹层面的高层控制；动力学模型在运动学之上引入质量、转动惯量与轮端力矩，刻画机器人对加速度与控制输入的响应能力，用于动力层面的精细控制；统一状态-输入定义则规定了所有控制器共用的状态向量与控制向量结构，从而保证 PID、LQR、MPC 在接口层面完全一致。

**图 2** 建模模块流程图

如图 2 所示，建模模块从差速驱动结构的几何关系出发，依次完成轮速-车体速度映射、位姿微分方程推导以及车体质量-转动惯量参数定义，最后将运动学与简化动力学整合为统一的连续时间状态空间表示。设计上之所以同时保留运动学与动力学模型，是因为运动学层面的描述简洁直观，便于 PID 与 LQR 在工作点附近线性化；而动力学层面的描述则为 MPC 中加速度约束与轮速约束的处理提供基础，并使仿真模型在物理意义上更接近实物。模块对外仅暴露"状态向量、控制向量、连续/离散动力学函数"三组接口，对内屏蔽具体推导细节，符合高内聚-低耦合的设计原则。

## 3.3 仿真平台模块设计

仿真平台模块的核心功能定位是：基于 MuJoCo 物理引擎，构建一个**可重复、可观测、易扩展**的差速驱动机器人测试环境，为三类控制器提供统一的测试基础。该模块包括**机器人 MJCF 模型**、**实验场景配置**、**传感器接口**和**控制接口**四个子组件。机器人 MJCF 模型按照建模模块给出的几何与物理参数描述车体、车轮与轮地接触特性；实验场景配置定义地面、坐标基准以及典型测试任务（直线、转向、圆形/8 字轨迹等）；传感器接口在仿真中暴露车体位姿、线/角速度、轮速等状态量；控制接口则接收上层控制器输出的车体期望速度或左右轮力矩，并将其作用于 MuJoCo 执行器。

**图 3** 仿真平台架构图

如图 3 所示，仿真平台模块按"模型层 → 物理仿真层 → 接口层"的纵向结构组织：模型层完成 MJCF 文件加载与参数注入；物理仿真层由 MuJoCo 内核负责积分求解；接口层则封装为 Python 端的状态读取/控制写入函数，向上为控制算法模块提供与实物机器人形式一致的接口。该模块的设计重点有三：其一，仿真步长与控制周期严格分离，以保证物理求解稳定性的同时支持不同频率的上层控制；其二，所有传感器与控制接口均以"状态向量、控制向量"形式对外暴露，与建模模块中的接口约定保持一致，从而使任意控制器都能即插即用；其三，实验场景配置以可配置文件形式管理，便于直接复用于多个工况实验，避免重复搭建。这样的分层设计既隔离了仿真器内部细节，又最大限度提高了实验可复现性与可扩展性。

## 3.4 控制算法模块设计

控制算法模块是本研究的核心，其功能定位是为差速驱动机器人提供 PID、LQR 与 MPC 三类典型控制策略，并通过统一的输入-输出接口接入仿真平台与实物系统。所有控制器共享同一组状态量（位姿与速度）和同一组控制量（车体期望线速度与角速度），并对外提供形式一致的 `step(state, reference) → control` 接口，使三类控制器可以在不改动外部环境的前提下相互替换，确保对比实验的公平性。

**图 4** 三类控制器统一接口框图

### 3.4.1 PID 控制器设计思路

PID 控制器作为对照基线，采用经典的**双环串级结构**：外环以位姿误差为输入，输出期望线速度与角速度参考；内环则以速度误差为输入，直接输出作用于左右轮的控制量。选择双环结构的原因在于，差速驱动机器人在工程上常将位姿控制和速度控制解耦处理，这样既符合工程惯例，又便于通过分通道独立调参实现稳定的基础控制。PID 控制器的优势在于实现简单、调试经验成熟，可作为评价 LQR 与 MPC 性能提升幅度的基准参考；其局限则在于不能显式利用系统模型与约束信息。

### 3.4.2 LQR 控制器设计思路

LQR 控制器以差速驱动机器人在期望轨迹工作点附近的线性化模型为基础，将位姿与速度误差作为状态量，构建标准的状态空间形式，并通过求解代数 Riccati 方程得到最优状态反馈律。设计上采用基于参考轨迹的"误差动力学"建模方式，使 LQR 自然地服务于轨迹跟踪任务，而不是简单的镇定问题。权重矩阵 $Q, R$ 的选取被设计为对外暴露的可调超参数，便于体现"快收敛"与"低控制能耗"两种偏好对系统响应的影响。LQR 控制器在结构上比 PID 更系统地利用了模型信息，可作为最优控制范式的代表与 PID、MPC 进行横向对比。

### 3.4.3 MPC 控制器设计思路

MPC 控制器在 LQR 所用线性化模型的基础上进一步引入约束条件与有限预测时域。在每个控制周期，MPC 求解一个带约束的二次规划问题，得到未来若干步的最优控制序列，并仅执行其中第一步控制输入。设计上着重处理三类约束：状态量约束（轨迹跟踪误差范围）、控制量约束（车体线/角速度上下限）以及控制增量约束（输入平滑性）。MPC 控制器的设计目标不在于追求理论上的最优性，而是体现出"显式建模 + 显式约束"在差速驱动机器人轨迹跟踪任务中的实际优势。求解器层面采用工程上较为成熟的 QP 方案，以保证控制周期内可稳定求解。

## 3.5 实物控制系统设计

实物控制系统模块的功能定位是将仿真平台中验证通过的控制逻辑迁移到差速驱动机器人实物平台上，并向上延续仿真接口的形式，向下负责底层电机驱动与传感器读取。模块由**上位控制层**、**通信链路**和**底层驱动层**三个子组件构成。上位控制层在 Jetson 等计算单元上运行，复用控制算法模块中的同一份控制器实现，输入为机器人状态量、输出为车体期望线/角速度；通信链路负责将控制指令下发至机器人下位机；底层驱动层在下位机或电机驱动器上完成期望速度到左右轮 PWM/电流控制的转换，并将编码器、IMU 等传感器读数回传至上位机。

**图 5** 仿真到实物迁移路径图

如图 5 所示，仿真到实物的迁移以**控制律共享 + 接口对齐**为核心：控制器代码在仿真与实物两端使用同一实现，仅通过外层适配函数切换状态读取与控制写入的来源；模型参数（轮距、质量、转动惯量等）作为可配置项注入，使仿真模型与实物保持参数一致；上层控制频率根据下位机能力进行降采样，但控制律本身不做修改。这一设计有助于将仿真与实物之间的差异隔离到"参数偏差"与"通信延迟"两类有限因素中，便于在第 4 章中进行有针对性的差异分析。

## 3.6 本章小结

本章给出了本研究系统的总体架构与各模块的设计思路。整体系统由建模、仿真平台、控制算法和实物控制系统四个模块构成，模块之间通过统一的状态-输入接口连接，确保 PID、LQR、MPC 三类控制器可以在仿真与实物两端无缝切换。在控制算法层面，PID 采用双环串级结构作为基线；LQR 基于误差动力学的线性化模型实现最优反馈控制；MPC 在 LQR 的模型基础上引入显式约束与滚动优化机制。仿真平台与实物控制系统在接口与控制律层面保持一致，为后续仿真-实物对比分析奠定基础。在此设计框架下，第 4 章将给出各模块的具体实现以及多工况下的实验结果。

本文围绕差速驱动移动机器人的运动仿真与控制问题，主要完成了以下工作：

1. 建立了差速驱动机器人的运动学与动力学模型，明确了状态变量与控制输入；
2. 基于 MuJoCo 搭建了可复用的差速驱动机器人仿真平台；
3. 实现了 PID、LQR、MPC 三类控制器，并完成了在统一仿真环境中的部署；
4. 通过直线、转向、轨迹跟踪等工况完成了三类控制器的量化对比；
5. 完成了仿真到实物机器人的迁移与基础验证实验。

## 5.2 不足与展望

**当前不足**：
- MPC 控制器在嵌入式平台上的实时性受限于求解器性能；
- 实物模型参数辨识精度有限，存在仿真-实物偏差；
- 暂未引入感知模块，轨迹由外部给定。

**未来工作**：
- 引入感知-规划-控制一体化框架（如结合 SLAM 与局部规划）；
- 探索基于学习的 MPC（learning-based MPC）以提升实时性；
- 在更复杂的非完整约束场景下扩展实验。
